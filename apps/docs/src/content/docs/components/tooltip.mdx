# Tooltip

Le tooltip TDS est CSS pur : la bulle s'affiche via `:hover` et `:focus-within`, sans
JavaScript. Cette simplicité a une contrepartie assumée : sur écran tactile, il n'existe pas
d'équivalent fiable au survol, donc l'information de la bulle ne doit **jamais** être la seule
source d'une information essentielle.

---

## Importation

```css
@import "@tds-tchad/core/components/tooltip/tooltip.css";
```

```typescript
import { tooltip } from '@tds-tchad/core/components/tooltip';
```

---

## Exemple

```html
<span class="tds-tooltip" tabindex="0" aria-describedby="tooltip-numero">
  Numéro de dossier
  <span class="tds-tooltip__bubble" id="tooltip-numero" role="tooltip">
    Identifiant unique attribué à la création de votre demande
  </span>
</span>
```

---

## Accessibilité

<Aside type="caution" title="Limitation connue — ne pas utiliser pour de l'information critique">
  Ce tooltip CSS pur ne s'affiche pas au simple tap sur écran tactile (pas d'équivalent CSS au
  `:hover`). Si l'information est indispensable à la compréhension ou à la complétion d'une
  tâche, ne la mettez **pas uniquement** dans un tooltip — affichez-la en clair ou dans un texte
  d'aide visible (`tds-helper-text`).
</Aside>

<Aside type="tip" title="Règles critiques">
  - L'élément déclencheur doit être focusable (`tabindex="0"` s'il ne l'est pas déjà nativement, ex. un `<span>`) pour que `:focus-within` fonctionne au clavier
  - La bulle a `role="tooltip"` et est référencée par `aria-describedby` sur le déclencheur
  - Contenu de la bulle : bref, complémentaire — jamais une action ou un lien (un tooltip n'est pas interactif)
</Aside>
