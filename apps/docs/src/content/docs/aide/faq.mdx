# Questions fréquentes

## TDS est-il officiellement adopté par l'État tchadien ?

Pas encore. TDS est actuellement une initiative indépendante et communautaire. Le projet vise
une adoption institutionnelle, mais ne se présente pas comme une norme officiellement
homologuée. Voir [Mission](/about/mission/) pour le détail du statut du projet.

## Puis-je utiliser TDS dans un projet non-gouvernemental ?

Oui. TDS est publié sous licence MIT — libre d'utilisation pour tout projet, y compris privé
ou commercial, sans obligation d'attribution.

## Comment TDS se compare-t-il au DSFR ou au GOV.UK Design System ?

TDS s'en inspire directement (voir [Mission](/about/mission/#inspirations)) mais reste un
projet distinct, adapté au contexte tchadien : palette aux couleurs nationales, support de
l'arabe en plus du français, portée réduite pour l'instant (17 composants contre plusieurs
dizaines pour le DSFR).

## Le package npm est-il publié ?

Pas encore — `@tds-tchad/core` n'est pas publié sur le registre npm public. Pour l'instant,
utiliser le repo directement (`pnpm add github:wargafu/TDS`) ou cloner le monorepo. La
publication npm effective fait partie de la [roadmap](/communaute/roadmap/) v1.0.

## Comment signaler un bug ou une régression d'accessibilité ?

Ouvrir une issue sur [GitHub](https://github.com/wargafu/TDS/issues) avec une description du
bug et si possible les étapes de reproduction. Voir [Contribution](/communaute/contribution/)
pour le format attendu.

## Puis-je proposer un nouveau composant ?

Oui, mais un nouveau composant nécessite une RFC (discussion ouverte) avant implémentation —
voir la section "Types de contributions acceptées" de la page
[Contribution](/communaute/contribution/).

## TDS fonctionne-t-il sans JavaScript ?

Presque tous les composants, oui — c'est un principe de conception central. Seuls Modal et Tabs
nécessitent un minimum de JavaScript côté consommateur (documenté sur leurs pages respectives),
car il n'existe pas de mécanisme HTML/CSS natif pour piloter leur état.
